HTML

  • Used to create the structure of a webpage.
  • Adds content to pages with text elements, image elements, header elements, and list elements.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The footer element at the bottom of the page usually contains author, copyright, contact information, sitemap, and navigation menu.

CSS

  • Cascading style sheets defines colors, aesthetic, visual layout, and organization.
  • Inline css styles apply to a specific html tag using a style attribute with a css rule to style particular page element, only style an element directly in html file.
  • Internal style sheet contains css rules for the webpage in the head section of the html file.
  • External style sheet contains css rules in a seperate file.
  • Seperation of concerns is the idea that each section of code should have its own responsibility.
  • Declarations contain two important components: css property we want to apply and the value of that property.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS box model states every html element is a box. Each box consists of margins outside the element, the elements borders, padding and content.
  • * symbol is used to apply a rule to all elements visable on a page.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout [branch name]: switches to specified branch
  • git add -A adds changed files in our working branch directory to the staging area of git to commit all files in the working branch
  • git commit -m "[descripted message]": commit your staged content as a new commit snapshot
  • git pull: fetch and merge any commits from the tracking remote branch

JavaScript

  • A variableis a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script
  • if condition is a blockof code to be executed if a condition is truthy
  • === strict equality operater checks to see if two values are equal and returns a Boolean result true if the values are equal and false if the values are not equal
  • array is a group of related data inside a single variable but are not limited to contain only a single type of data such as strings, numbers, and so on.
  • for loop uses a predictable pattern of indices to perform a task on ALL items in array by allowing single code block to be executed over and over
  • function is a set of instructions that tells the computer how to preform a certain task and do not automatically execute when the java file is run
